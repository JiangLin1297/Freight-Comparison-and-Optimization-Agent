# 优先级自动识别功能使用指南

## 功能概述

系统现在能够自动识别用户输入中的优先级偏好，并自动切换到相应的推荐模式：
- **时效优先模式**：优先推荐最快的方案
- **成本优先模式**：优先推荐最便宜的方案
- **均衡模式**：使用综合评分推荐（默认）

---

## 支持的关键词

### 时效优先（priority: "time"）

当用户输入包含以下词汇时，系统会自动切换到时效优先模式：

| 类型 | 关键词示例 |
|------|-----------|
| 直接表达 | 越快越好、最快、最快速度、尽快、尽早 |
| 时间优先 | 时间优先、时效优先、速度优先 |
| 紧急需求 | 加急、紧急、特急、急、急需、很赶时间 |
| 到达要求 | 最快到达、最短时间、时效第一、能多快就多快 |
| 期望表达 | 越早越好、马上要、立即、马上 |

### 成本优先（priority: "cost"）

当用户输入包含以下词汇时，系统会自动切换到成本优先模式：

| 类型 | 关键词示例 |
|------|-----------|
| 直接表达 | 最省钱、越便宜越好、最便宜、最低价 |
| 成本优先 | 成本优先、价格优先、省钱、费用最低 |
| 经济实惠 | 经济实惠、性价比高、划算 |
| 预算考虑 | 预算有限、控制成本、能省则省、省钱为主 |
| 期望表达 | 便宜点的、便宜一些、少花点钱、价格低的 |

### 均衡模式（priority: null）

当用户没有明确的优先级偏好时，系统使用默认的均衡模式，基于综合评分（成本40% + 时效30% + 服务30%）进行推荐。

---

## 使用示例

### 示例1：时效优先

**用户输入：**
```
从大连运100kg货物到厦门，越快越好
```

**系统识别：**
- 起运港：PORT08（大连）
- 目的港：PORT09（厦门）
- 重量：100kg
- 优先级：time（时效优先）

**系统行为：**
1. 自动切换到时效优先模式
2. 推荐运输天数最少的方案
3. 显示提示："已识别为时效优先模式，将优先推荐最快的方案"

---

### 示例2：成本优先

**用户输入：**
```
从上海运50kg到深圳，最省钱的方案
```

**系统识别：**
- 起运港：PORT02（上海）
- 目的港：PORT03（深圳）
- 重量：50kg
- 优先级：cost（成本优先）

**系统行为：**
1. 自动切换到成本优先模式
2. 推荐总成本最低的方案
3. 显示提示："已识别为成本优先模式，将优先推荐最便宜的方案"

---

### 示例3：带时效要求的成本优先

**用户输入：**
```
从天津运200kg到香港，3天内到，越便宜越好
```

**系统识别：**
- 起运港：PORT07（天津）
- 目的港：PORT10（香港）
- 重量：200kg
- 最大天数：3天
- 优先级：cost（成本优先）

**系统行为：**
1. 先筛选出3天内能到达的方案
2. 在这些方案中推荐成本最低的
3. 显示提示："已识别为成本优先模式，将优先推荐最便宜的方案"

---

### 示例4：均衡模式

**用户输入：**
```
从广州运100kg到厦门
```

**系统识别：**
- 起运港：PORT04（广州）
- 目的港：PORT09（厦门）
- 重量：100kg
- 优先级：null（均衡模式）

**系统行为：**
1. 使用默认的均衡模式
2. 基于综合评分（成本40% + 时效30% + 服务30%）推荐
3. 显示推荐方案的评分详情

## 技术实现

### 后端实现

1. **LLM提示词优化**
   - 在 `llm_service.py` 中添加详细的优先级识别规则
   - 支持中文语义理解，识别各种表达方式

2. **解析结果扩展**
   - `parse_order` 方法返回 `priority` 字段
   - `chat_with_tools` 方法在工具调用时传递 `priority` 参数

3. **评分算法支持**
   - 根据 `priority` 自动调整权重配置
   - 时间优先：成本20% + 时效60% + 服务20%
   - 成本优先：成本60% + 时效20% + 服务20%

   - 显示识别出的优先级标签
   - 高亮显示优先级模式

2. **表单组件**
   - 支持接收 `priority` 字段
   - 自动配置权重滑块

3. **AI助手组件**
   - 显示工具调用的优先级参数
   - 展示优先级识别结果

## 常见问题

### Q1: 如果用户同时输入了"尽快"和"便宜"怎么办？

A: 系统会根据上下文判断主要意图。如果无法确定，会优先识别第一个出现的优先级关键词。

### Q2: 优先级识别的准确率如何？

A: 系统使用LLM进行语义理解，能够识别90%以上的常见表达。对于模糊或复杂的表达，可能会识别为均衡模式。

### Q3: 可以手动覆盖自动识别的优先级吗？

A: 可以。用户可以在表单中手动调整权重配置，覆盖自动识别的结果。

### Q4: 优先级识别支持方言或口语化表达吗？

A: 支持常见的口语化表达，如"急死了"、"能省就省"等。对于特殊方言，建议使用标准表达。
